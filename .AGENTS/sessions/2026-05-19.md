# Session Log - 2026-05-19

## 2026-05-19 - Compact command agent routine

- Added an explicit `memory-seed compact` agent routine to `.AGENTS/agent-rules.md` and the bundled seed copy.
- Rule: when the user asks to run `memory-seed compact`, agents should run it from the repository root, read the generated summary, report key session facts, and recommend durable facts for promotion without automatically editing durable memory files.
- Control-plane `memory-system-version` remains `1.4`; this is additive workflow guidance.

## 2026-05-19 - Consolidation review triggers

- Added numeric and event-based consolidation review triggers to `.AGENTS/agent-rules.md` and the bundled seed copy.
- Triggers include more than three meaningful session entries since last consolidation, major workflow/architecture/release changes, roughly 2,000 words / 8,000 tokens of notes or compact output, release/publish/migration/bootstrap/security/refactor events, or stale durable memory.
- Clarified that triggers require review, not automatic edits; durable promotion remains judgment-based.

## 2026-05-19 - CLI command behavior documentation

- Expanded `README.md` with existing-project behavior for `version`, `doctor`, `init`, `update`, and `compact`.
- Documented known behavior: `update --dry-run` lists all control-plane targets rather than only files that would actually change, and `init --force` rewrites all bundled seed files as a reinstall operation.
- Updated `update --dry-run` CLI help text to say it lists control-plane targets without writing.

## 2026-05-19 - Local semantic cache core

- Added `memory_seed.semantic_cache` as an importable core module for heading-aware session chunk extraction and ranked local memory retrieval.
- Implemented typed `MemoryChunk` and `RankedMemoryChunk` containers, optional `EmbeddingProvider` integration, lexical-first scoring, cosine similarity, recency decay, and structural-query decay reduction.
- Added tests for Markdown heading chunking, metadata extraction, lexical overrides, optional semantic scoring, embedding fallback, recency floor behavior, and session ranking.
- Rollback checkpoint before implementation: commit `ea06d8f` (`Document memory routines and CLI behavior`).

## 2026-05-19 - MCP memory search adapter

- Added a dependency-light `memory_seed.mcp_server` stdio JSON-RPC adapter over the semantic cache core.
- Exposed `memory_search` for ranked, source-linked session memory results and `memory_get_chunk` for exact chunk retrieval.
- Added `memory-seed-mcp --stdio` as the console entry point.
- Added human-validatable MCP tests that assert expected queries return specific dated session entries with source paths, line ranges, heading paths, scores, matched fields, and excerpts.
