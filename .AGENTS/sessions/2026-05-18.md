# Session Log - 2026-05-18

## 2026-05-18 - CLI update command

- User approved `update` as the command keyword for refreshing reusable control-plane documents.
- Added `memory-seed update` as a CLI command that refreshes only the reusable seed files: `AGENTS.md`, `CLAUDE.md`, `GEMINI.md`, `.AGENTS/agent-rules.md`, and `.AGENTS/project-bootstrap.md`.
- `update` uses each control-plane file's `memory-system-version` YAML field to decide whether it is current.
- `update` backs up replaced control-plane files under `.AGENTS/backups/<timestamp>/`, restores missing reusable seed files, skips files already on the current control-plane version, and leaves generated project memory files such as `context.md`, `index.md`, `style.md`, and `sessions/` unchanged.
- Added `memory-seed update --dry-run` to preview the planned control-plane refresh without writing files.
- Bumped the Python package version to `1.3.1`; the reusable control-plane `memory-system-version` remains `1.3`.

## 2026-05-18 - Orchestration levels and discarded dashboard direction

- Implemented conservative orchestration levels in the reusable agent rules, from direct single-agent work through research-first human checkpoints.
- Added token and model budget guidance that defaults to the lowest safe orchestration level and treats model choice as economy, standard, or frontier capability tiers.
- Discarded the local HTML dashboard direction before commit because the user clarified that an HTML frontend clutters Memory Seed's purpose.
- Archived reusable control-plane version `1.3` under `.AGENTS/archive/1.3/`.
- Bumped reusable control-plane `memory-system-version` to `1.4` and Python package version to `1.4.0`.

## 2026-05-18 - Semble integration into control plane

- Researched Semble (github.com/MinishLab/semble): a fast code search tool for agents using hybrid semantic + BM25 retrieval; ~98% fewer tokens than grep+read; runs on CPU with no API keys.
- Decision: not worth building a Markdown-optimised variant — Memory Seed files are intentionally small; the `compact` command is the right response to corpus growth, not search.
- Added a `## Code Search` section to `.AGENTS/agent-rules.md` establishing the tool hierarchy: `semble search` → `grep` → full-file read, scoped to software/library/API/tool projects.
- Added a Semble `AGENTS.md` snippet to `.AGENTS/project-bootstrap.md` Step 4, generated for code projects during bootstrap.
- Added a reference to the Semble snippet in Step 7 profile-specific guidance for "Production app/API/SaaS" and "Library or package" profiles.
- Control-plane `memory-system-version` remains `1.4`; changes are additive guidance, not structural.

## 2026-05-18 - Compact command and v1.5.0 release

- Implemented `memory-seed compact` CLI command: reads `.AGENTS/sessions/*.md`, produces a structured Markdown summary (headings + full text) for agent-driven consolidation.
- Flags: `--days N` (default 7), `--all`, `--output path`.
- Design: CLI summarises, agent judges what to promote into durable files. No automated writes.
- Added 5 unit tests for compact (headings, date filter, `--all`, empty sessions, non-date filenames).
- Added Windows UTF-8 stdout fix in `cli.py` for session files containing Unicode.
- Bumped package version to `1.5.0`.
- Published via `gh release create v1.5.0` to follow the convention of release-triggered workflow runs.
- Decision: always publish via GitHub Release (`gh release create`), not `gh workflow run`, so the Actions UI shows a labelled version.
- Promoted publishing convention, compact command status, and current CLI commands into `context.md`.
