### Summary

- Added MCP server registration to `memory-seed init` and `update` for Claude Code, Cursor, and Gemini.
- Added PATH detection to `memory-retrieval-check.py`: if `memory-seed-mcp` is not on PATH (e.g. after `uvx memory-seed init`), the hook now outputs an actionable install message instead of instructing the agent to call a tool that isn't available.

### Decisions

#### D1 - Write MCP config unconditionally, no PATH check at init time

- D: `init_project` writes MCP server config for all three vendors unconditionally, without checking if `memory-seed-mcp` is on PATH first.
- R: If the user ran `memory-seed init`, the binary shipped in the same package and should already be on PATH. Checking at init time adds complexity for a failure case that can't happen through normal persistent installation.
- A: Check PATH at init and warn — rejected because the check is fragile (binary could disappear later) and doesn't cover the `uvx` case anyway.
- F: `memory_seed/core.py`.

#### D2 - Detect missing binary in the retrieval hook, not at init time

- D: `memory-retrieval-check.py` uses `shutil.which("memory-seed-mcp")` to detect whether the server is on PATH, and surfaces a `uv tool install memory-seed` instruction when it isn't.
- R: The hook fires at prompt time in the user's editor — it's the right moment to surface a fix. The `uvx` case (init ran ephemerally, binary not persisted) is the primary motivation.
- F: `memory_seed/seed/.memory-seed/hooks/memory-retrieval-check.py`, `.memory-seed/hooks/memory-retrieval-check.py` (byte-identical).

#### D3 - Cursor MCP uses separate .cursor/mcp.json, not hooks.json

- D: Cursor's MCP registration goes into `.cursor/mcp.json`; hooks stay in `.cursor/hooks.json`.
- R: Cursor treats hook config and MCP server config as separate files. Merging them into `hooks.json` would break Cursor's MCP discovery.
- F: `memory_seed/core.py` (`_merge_cursor_mcp`).

#### D4 - Claude MCP entry includes "type": "stdio"; Cursor and Gemini omit it

- D: Claude Code's `mcpServers` entry includes `"type": "stdio"`; Cursor and Gemini entries omit it.
- R: Claude Code's schema requires the `type` field. Cursor and Gemini infer stdio from the command shape and reject or ignore unknown fields.
- F: `memory_seed/core.py`.

### Implementation

- `memory_seed/core.py`: added `_MCP_SERVER_COMMAND`, `_MCP_SERVER_ARGS`, `_MCP_SERVER_KEY` constants; added `_merge_claude_mcp`, `_merge_cursor_mcp`, `_merge_gemini_mcp` functions; appended all three to the `hook_merges` tuple in both `init_project` and `update_project`.
- Both `memory-retrieval-check.py` copies: added `import shutil`; replaced single `reminder` assignment with `shutil.which` branch.

## 2026-05-29 14:07 - Fix hook and MCP upgrade behavior (upsert) (upsert, not add-only)

```yaml
entry_id: ms-7b3f1e92

agent_type: claude-sonnet-4-6

```

### Summary

- Made all vendor config merge functions upsert (update-or-insert) instead of add-only.
- Previously: if a hook command changed between versions, `update` would append a new entry alongside the old one, causing both to fire.
- Now: merge functions identify their own entries by script filename (stable across command-string changes) and replace a stale command in place.

### Decisions

#### D1 - Script filename as the stable identifier for hook upsert

- D: Detect "our" hook entry by checking if the command string contains the script filename (e.g., `session-log-check.py`), not by exact command match.
- R: The script filename doesn't change between versions. Flags added to the command (the most likely version bump) would cause an exact-match check to miss, appending a duplicate. The filename check survives that.
- A: Version-stamp comments inside hook entries — rejected, would require non-standard comment fields in JSON.
- F: `_merge_grouped_hook`, `_merge_cursor_event_hook` (both now take `script_name` param).

#### D2 - Redirect four direct session-log functions to the helpers

- D: `_merge_claude_hook`, `_merge_codex_hook`, `_merge_gemini_hook`, `_merge_cursor_hook` previously duplicated the grouped/cursor-event logic inline. Replaced them with one-line delegations to the helpers.
- R: Eliminates four copies of the same upsert logic. All eight hook functions now share exactly two implementations.
- F: `memory_seed/core.py` — four functions reduced to thin wrappers.

#### D3 - MCP upsert: overwrite if command matches, skip if different command owns the key

- D: MCP merge functions compare the full expected dict. If the key exists with a different `command`, leave it alone (user may have a different server named `memory-seed`). If the command matches but args differ, overwrite.
- R: Protects user customization while still correcting stale args from our own prior installs.
- F: `_merge_claude_mcp`, `_merge_cursor_mcp`, `_merge_gemini_mcp`.
