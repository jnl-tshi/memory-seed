### Summary

- Updated MCP memory parsing so session entries are the default coherent chunk unit and `entry_id` becomes the entry-level `chunk_id`.
- Added optional section granularity with ids such as `entry_id#decisions/d1-title`, while preserving parent entry metadata on section results.
- Added operating-mode rules that tell agents to use `memory_search` and `memory_get_chunk` for relevant history, treat current files as authority, and ask the user when historical and current design decisions conflict ambiguously.

### Decisions

#### D1 - Make entry chunks the default MCP memory unit

- D: Parse `##` session entries as the default chunk boundary and use the entry YAML `entry_id` as `chunk_id`.
- R: A `##` entry is the coherent memory block carrying shared metadata, rationale, implementation, and validation.
- A: Rejected default heading-level chunking because it can separate a decision from its rationale and validation.
- F: `memory_seed/semantic_cache.py`, `memory_seed/mcp_server.py`, parser/MCP tests.
- T: `python -m unittest tests.test_semantic_cache tests.test_mcp_server tests.test_mcp_validation tests.test_session_schema`; `python -m unittest discover -s tests`.

#### D2 - Preserve optional section granularity

- D: Add `granularity: "entry" | "section"` to `memory_search`, with `entry` as default and `section` available for long or multi-topic entries.
- R: Entry-level retrieval should be the normal agent-memory behavior, but narrow searches are useful for large plan-mode logs.
- A: Rejected removing section retrieval entirely because some projects may need more targeted search.
- F: MCP tool schema, `memory_get_chunk` lookup, section-id tests.
- T: Section smoke test returned `ms-db2d715c#decisions/d1-use-draft-for-compact-decision-records`.

#### D3 - Ask the user on unresolved history conflicts

- D: Current `index.md`, `policy.md`, active skills, and source/config files are active authority; session history is evidence and rationale. If history conflicts with current files and no clear newer supersession boundary exists, agents must ask the user before changing durable behavior.
- R: The user wanted ambiguity handled explicitly rather than inferred from partial or conflicting memory.
- A: Rejected blindly following either older history or current files when design intent is unclear.
- F: `.memory-seed/agent-rules.md`, `.memory-seed/project-bootstrap.md`, seed copies, session-schema tests.
