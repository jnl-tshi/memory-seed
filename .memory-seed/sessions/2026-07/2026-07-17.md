### Summary

- Hardened Memory Trace's derived SQLite cache so two local server processes can rebuild the same workspace projection without producing a Windows file-lock 500.

### Decision

- D: Use a short-lived OS-level writer lease per primary cache path. A waiting process rechecks the winner's snapshot after it obtains the lease; when the lease remains unavailable after a bounded wait, it builds an isolated temporary projection instead of contending for the shared cache.
- R:
  - The observed failure was an `os.replace` denial during concurrent rebuild, not a Markdown or UI data error.
  - Markdown remains authoritative; both the shared SQLite cache and emergency temporary projections are disposable derived state.
- A: Rejected making unique temporary caches the normal default because it hides contention, duplicates warm caches, and leaves normal multi-process coordination unresolved. The session scaffolder could not create this new dated file in the OneDrive worktree, so this append uses the canonical generated entry ID.

### Decisions

#### D1 - Ship `link add`, refuse any non-newest source

- D: Implement `memory-seed link add <target> [--from]` writing one `related_entries` edge from the
  newest entry to an older one. Forward-only, idempotent, YAML-only, `links check`-gated. Any source
  that is not the newest entry is refused at the writer, not just at the CLI.
- R:
  - Adding to the entry being authored does not rewrite history, so it is clean under Invariant #2,
    and the plan itself calls this "the normal path because it does not rewrite history".
  - Enforcing in the library (not the CLI) means the guard cannot be bypassed by a future caller,
    which matters because the refused path is exactly the unratified backfill.
- A: Rejected implementing the plan whole. Rejected putting the newest-entry check only in the CLI.
- F: `memory_seed/semantic_cache.py`, `memory_seed/cli.py`, `tests/test_semantic_cache.py`,
  `CHANGELOG.md`, `docs/2_Todo/related-entries-p2-mutation-plan.md`.
- T: `pytest tests/` 526 pass; core CI invocation 287 pass; `memory-trace/tests` 146 pass. Eight new
  cases cover append, key-creation, idempotency, self/forward/unknown refusal, the Invariant #2
  refusal, file-ending preservation, and unterminated-block refusal.

#### D2 - The P2 historical backfill is blocked pending a constitutional ruling

- D: Do not build `link backfill`. Record it in the plan as an open user decision with three options:
  reject it in favour of the shipped evolution-edges seeding pass, amend Invariant #2 for a
  metadata-only carve-out, or record a reading that YAML curation was never "rewriting the past".
- R:
  - The plan was approved 2026-07-05; the Constitution was ratified 2026-07-14. The sign-off predates
    the invariant and cannot authorise it retroactively.
  - Backfill mutates existing entries' YAML, which collides with Invariant #2 ("never rewrite or
    delete"). Constitution §11: a proposal conflicting with a live invariant "is rejected or must
    first amend the invariant - it cannot silently override it".
  - The evolution-edges seeding pass already adds edges to history by writing *new* entries, so the
    capability may not need the invariant bent at all.
- A: Rejected building it on the 2026-07-05 approval. Rejected silently dropping it from the plan.
- F: `docs/2_Todo/related-entries-p2-mutation-plan.md`.

#### D3 - Fixed a crash found by reviewing the change, not by the tests

- D: Bound the closing-fence search to the source entry and refuse an unterminated yaml block.
- R: The first implementation scanned the whole file for the closing fence: an entry whose fence ran
  to EOF raised `IndexError`, and a stray fence in a later entry could have spliced metadata into
  another entry's prose. Every test passed while this was broken - the review caught it, so it is
  now pinned by a regression case.
- A: Also removed dead CRLF-preservation code: `read_text_file` uses universal newlines and
  `write_text_file` re-emits canonical LF, so the newline detection could never fire.
- F: `memory_seed/semantic_cache.py`, `tests/test_semantic_cache.py`.
