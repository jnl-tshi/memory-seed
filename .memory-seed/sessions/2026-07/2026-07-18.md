### Decision

- D: Two additions that make the Trail readable and searchable. (1) A relationship **legend** in the
  viewbar — dashed keys for replaces/evolves/related in their edge colours, so the arcs shipped in 4a can
  be decoded. (2) **Search as a function over the Trail**: `TrailWorkspace` takes the live `query`;
  matching rows (client-side substring over title/branch/entry-id within the visible window) keep full
  presence and gain a marker dot, non-matching rows and dots dim, and the viewbar shows a live match
  count. Clearing the query restores everything.
- R:
  - The arcs landed in 4a with colour meaning the user had no way to read — the legend is the smallest
    high-value complement. Search-dimming turns the Trail into a working surface (find the entry, see it
    in lineage context) rather than a static view.
  - I chose a **client-side substring filter over the visible window** rather than reusing the semantic
    search dropdown's server results: the dropdown returns a relevance-capped top-N, which would dim many
    visible rows that *do* contain the term (reads as broken), whereas the client filter is instant,
    complete for what's on screen, and honest ("these visible rows contain your term"). The semantic
    dropdown still works for navigation; the two are complementary.
- A: Rejected server `matchEntries` (top-N coverage gap on a recency-ordered window). Deferred continuity
  lanes (rename/migration/removal) to Slice 5b — they need the union-find model + glyphs + a left-zone
  offset and are low-frequency (4/458 nodes), so they don't belong in the same cut as the common-path
  legend/search.
- F: `memory-trace/client/src/TrailWorkspace.tsx`, `App.tsx` (passes `query`), `styles.css`,
  rebuilt `memory_trace/static/react/*`, `docs/2_Todo/0_NEXT_STEPS.md`, `CHANGELOG.md`.
- T: `tsc --noEmit` clean; `vite build` clean; `test_static_serving.py` 9 pass; `docs index --check`

  match dots / "4 matches" in the meta, and the three sampled matched rows genuinely contain the term;
  clearing the query resets to 0 miss / 0 dots; no console errors.
