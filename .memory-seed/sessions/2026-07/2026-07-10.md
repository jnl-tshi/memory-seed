### Decisions

#### D1 - Strict UTF-8 decode in the git subprocess helpers

- D: Add `encoding="utf-8"` to `_git_text`, `_git_show_text`, and `find_trailer_commits`, and catch
  `UnicodeDecodeError` alongside `OSError`/`TimeoutExpired`.
- R: `subprocess.run(text=True)` without `encoding=` uses the Windows locale (cp1252), which crashed
  on non-ASCII session content; strict UTF-8 matches the codebase's `CANONICAL_ENCODING` convention,
  and the catch degrades a genuinely-corrupt file to the existing "unreadable → skip" path instead
  of crashing (mirrors `_merge_head_commits`).
- A: Chose strict over `errors="replace"` because the repo reserves `replace` for console stdio only;
  silent substitution would corrupt text that fuse compares for immutability and writes back.
- F: `memory_seed/core.py`.

#### D2 - Scope branch-side validation to the branch delta

- D: Add `_changed_session_paths()` using three-dot `git diff --name-only <base>...<branch>`, an
  `only_paths=` filter on `_entry_records_from_ref`/`_sidecar_records_from_ref`, and apply it to the
  branch-side enumeration in `session_fuse`; base-side stays full.
- R: The branch commit contains the entire base corpus, so whole-tree enumeration flagged unchanged
  legacy pre-schema files (no `entry_id`) as branch problems. "Branch-local" means files the branch
  changed. Three-dot (not two-dot) avoids surfacing base's post-divergence appends to shared dated
  files. Base-side must stay full for already-present dedup and sidecar-parent-on-base lookups.
  (User chose the changed-files approach over per-entry already-present skipping.)
- A: `_changed_session_paths` returns `None` (not an empty set) on git failure, and `session_fuse`
  blocks with an issue — a review finding: an empty set would silently filter out every branch entry
  and report success importing nothing.
- F: `memory_seed/core.py`, `tests/test_memory_seed.py`.
